# UAT Test Plan & Usage Steps — agentic-platform

**Audience:** QA, UAT testers, developers validating before production/SaaS  
**Scope:** Healing (multi-framework) + autonomous QA + healing-service  
**Out of scope for this UAT:** npm publish, production SaaS billing/auth

**Repo:** clone [agentic-platform](https://github.com/mmohansqaai/agentic-platform) or use local monorepo root.

---

## 1. Prerequisites

| Requirement | Version / note |
|-------------|----------------|
| Node.js | 20+ |
| npm | 9+ |
| Java (optional, Java client UAT) | 17+ + Maven |
| Browsers | `npm run install:browsers` |
| LLM keys (optional Tier 2+) | OpenAI or Anthropic — **service only**, not in tests |

```bash
git clone https://github.com/mmohansqaai/agentic-platform.git
cd agentic-platform
npm install
npm run install:browsers
```

**Build once:**

```bash
npm run build:healing-service
npm run build:autonomous-qa-sdk
npm run build:autonomous-test-agent
```

## 2. Usage steps (by role)

### A. Developer — healing only (Playwright, no server)

**Goal:** Broken locator self-heals in-process.

```bash
npm run test:plug-and-play
```

Or manually:

```bash
npm run install:plug-and-play-demo
export AUTO_HEAL_DISCOVER=1
export HEALING_AGENT_MODE=agentic
npm test --prefix examples/playwright-plug-and-play
```

**In test code:**

```typescript
import { enableHealing, healable } from 'ai-healing-sdk';

enableHealing(page, { healingEnabled: true, agentMode: 'agentic' });
await healable.click(page.locator('#intentionally-wrong'));
```

**Pass criteria:** Test passes; report shows `autoHeal.usedAutoGenerated: true` or healed strategy in attachments.

---

### B. DevOps — healing-service (cloud agent)

**Terminal 1 — start service:**

```bash
export HEALING_LLM_PROVIDER=mock   # UAT: no API key needed
npm run healing-service
# → http://localhost:3921
```

**Terminal 2 — run tests against service:**

```bash
export HEALING_SERVICE_URL=http://localhost:3921
export AUTO_HEAL_DISCOVER=1
export HEALING_AGENT_MODE=agentic
npm run test:plug-and-play
```

**Verify health:**

```bash
curl -s http://localhost:3921/health | jq .
# Expected: { "status": "ok", "service": "healing-service", ... }
```

### E. Multi-framework adapters (manual UAT)

**Cypress / Selenium JS** — use in a consumer project with `file:` dependency from monorepo, or after build:

```bash
npm run build:cypress    # or build:selenium
```

```typescript
// Cypress
import { enableHealing, healable } from 'ai-healing-cypress';
enableHealing({ healingServiceUrl: process.env.HEALING_SERVICE_URL });
await healable.click('#wrong-selector');

// Selenium JS
import { enableHealing, healable } from 'ai-healing-selenium';
enableHealing(driver, { agentMode: 'local' }); // or auto + HEALING_SERVICE_URL
await healable.click('#wrong-selector');
```

**Java:**

```bash
npm run build:java
# Use packages/ai-healing-java in Selenium Java project (see packages/ai-healing-java/README.md)
```

### F. Environment variables reference

| Variable | UAT typical value | Purpose |
|----------|-------------------|---------|
| `AUTO_HEAL_DISCOVER` | `1` | Enable healing after primary locator fails |
| `HEALING_AGENT_MODE` | `agentic` | Agent loop (Playwright SDK) |
| `HEALING_AGENT_MAX_ITERATIONS` | `3` | Max reflect iterations |
| `HEALING_SERVICE_URL` | `http://localhost:3921` | Route to cloud agent |
| `HEALING_AGENT_MODE` (adapters) | `auto` / `local` / `remote` | Cypress/Selenium adapter mode |
| `HEALING_LLM_PROVIDER` | `mock` | Service LLM (UAT without keys) |
| `RUN_UNIT_TESTS` | `1` | Enable `*.unit.spec.ts` |
| `RUN_AUTONOMOUS_CI` | `1` | Nova CI smoke gate |

```bash
npm install && npm run install:browsers
npm run build:healing-service && npm run build:autonomous-qa-sdk && npm run build:autonomous-test-agent

npm run test:plug-and-play:ci
```

```bash
# Terminal 1
```

## 6. Troubleshooting

| Symptom | Fix |
|---------|-----|
| `healing-service.unit` fetch failed | Start `npm run healing-service` first |
| Nova install fails | Run `npm run build:autonomous-qa-sdk` from root first |
| No healing in test | Set `AUTO_HEAL_DISCOVER=1` |
| Cypress/Selenium no heal | Set `agentMode: 'local'` or `HEALING_SERVICE_URL` |
| LLM tests skip/fail | Expected without keys; use `HEALING_LLM_PROVIDER=mock` on service |
